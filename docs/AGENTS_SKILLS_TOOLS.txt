Companion plan (phased work): docs/IMPLEMENTATION_PLAN_AGENTS_TOOLS.txt

This document lists sub-agents, bundled skills, core native tools, and SDK-style
plugins as wired in the repository. Runtime availability may still depend on
config, security policy, session tool allowlists, and environment.

Sources of truth in code:
  - Agents:        src/agent/factory.ts, core/types (SubAgentKind)
  - Skills:        src/skills/*/tool.ts, src/plugins/builtinTools.ts
  - Core tools:    src/tools/*.ts (toolRegistry.register / name: '…')
  - SDK plugin:    src/plugins/ScreenControl/manifest.json, src/core/pluginLoader.ts
  - MCP bridge:    src/plugins/mcpBridge.ts (names depend on MCP server config)
  - Naming in config: docs/TOOL_REGISTRY_NAMING.txt (allowlists, apex.json)
  - How to add features: docs/EXTENDING_APEX.txt

How to refresh (machine-readable lists)
  Run from the repository root:
    npm run docs:list-tools
  This prints sorted tool ids from src/tools, src/skills/*/tool.ts, and SDK
  plugin names (e.g. ScreenControl.get_ui_tree) from the ScreenControl manifest.
  Use it to diff against older output or to update the alphabetical sections in
  this file. Narrative sections (sub-agents, MCP notes, CLI) stay hand-edited.

  CI / drift check: committed snapshot is docs/tools-list.golden.txt. After tool
  changes, regenerate it with:
    node scripts/list-tools.mjs > docs/tools-list.golden.txt
  Verify with: npm run docs:check-tools (also covered by npm test).

1. SUB-AGENTS (SubAgentKind)
================================================================================
Kinds (union type): researcher | coder | system

  researcher — "Researcher Agent"
    Focus: gather context, read/search, consult memory, synthesize evidence.

  coder — "Coder Agent"
    Focus: inspect, edit, debug, execute technical tasks.

  system — "System Agent" (default)
    Focus: macOS power-user: OS actions, AppleScript/UI control, notifications,
    schedule-aware behavior, monitoring, device operations. System prompt also
    mandates get_today_schedule before availability/meeting suggestions and
    Spotify usage rules (see factory).

Routing (heuristic, AgentFactory.choose / chooseWithDiagnostics):
  • metadata.subAgentKind (session/TUI override) wins when set to researcher | coder | system.
  • Otherwise scored hints (phrases + word-boundary tokens) produce coder vs researcher scores;
    higher score wins; ties with both sides scoring prefer coder (legacy ordering); equal
    ambiguous scores → system. See src/agent/factory.ts (CODER_*/RESEARCHER_* lists).
  • Diagnostics: chooseWithDiagnostics → source, scores, confidence. Logged at debug in
    core/brain when the orchestrator wires chooseWithDiagnostics.
  • Eval golden: test/evals/golden.cases.json + test/evals/goldenEval.test.ts
    (routing, tool policy, negatives). Run: npm run test:eval or npm test.

Researcher tool policy (capability tier):
  • **coder** / **system**: full session tool allowlist (no shrinking by kind).
  • **researcher**: read-oriented isolation — tools with permission class **read**; **write** only
    for allowlisted memory/soul/consult names (see RESEARCHER_WRITE_ALLOWLIST in
    src/agent/subAgentToolPolicy.ts) or manifest category **memory**; **automation** and
    **destructive** are never included. Implementation: filterToolsForSubAgentKind.

Helper invocation (not a SubAgentKind enum value):
  • Tool: spawn_helper_agent — runs a separate orchestrator task for a sub-task
    (see src/tools/multiAgent.ts).

================================================================================
4. SDK PLUGIN — ScreenControl
================================================================================
Plugin id: ScreenControl (see src/plugins/ScreenControl/manifest.json)

Tools are registered with a plugin prefix in the tool registry, e.g.:
  ScreenControl.get_ui_tree
  ScreenControl.ui_click_element

(Exact string is pluginId + '.' + tool name; see src/core/pluginLoader.ts.)

================================================================================
5. MCP TOOLS (configuration-dependent)
================================================================================
MCP servers expose tools bridged into the registry with sanitized names such as:
  mcp_<serverId>__<toolName>
See src/plugins/mcpBridge.ts. There is no static list in-repo; it depends on
your MCP / apex.json (or equivalent) configuration.

================================================================================
6. CLI (NOT model tools)
================================================================================
Shell commands like `apex files preview` / `apex files apply` / `apex help` are
implemented in the CLI early-exit path and are not registered in toolRegistry.
